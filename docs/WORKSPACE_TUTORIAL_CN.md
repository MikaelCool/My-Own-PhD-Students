# My Own PhD Students 可视化工作台使用教程

这份文档只讲工作台怎么用

适用对象：
- 第一次使用可视化页面的人
- 想知道每个选项该怎么填的人
- 本地算力不够 想切到云服务器的人

## 1. 如何打开页面

在项目根目录执行：

```powershell
cd D:\codex\AutoResearchClaw
.\.venv\Scripts\python.exe -m researchclaw.cli serve --config .\config.arc.yaml --host 127.0.0.1 --port 8080
```

浏览器打开：

```text
http://127.0.0.1:8080/
```

如果页面没刷新到最新样式，按一次 `Ctrl + F5`

## 2. 页面结构怎么看

工作台主界面分成 4 个核心区：

- `Start Research`
  用来新建一个研究课题
- `Details`
  用来看项目当前状态 最新运行 关键工件和合同目标
- `Canvas`
  用来从结构图看阶段 工件 决策和回滚关系
- `Studio`
  用来看消息流 时间线 日志 以及控制运行

## 3. Start Research 怎么填

`Start Research` 现在分成 3 个子页：

- `基本信息`
- `研究策略`
- `资料与约束`

### 3.1 基本信息

`Project ID`

- 作用：项目唯一标识
- 建议怎么填：只用英文 小写 数字 中横线
- 示例：`llm-robustness-01`

`Project title`

- 作用：页面里显示的项目标题
- 建议怎么填：写给人看的名字
- 示例：`大模型鲁棒性与错误恢复研究`

`Primary request`

- 作用：这是整个研究的核心任务说明
- 应该填什么：
  - 你到底要研究什么
  - 成功标准是什么
  - 你希望系统交付什么
- 推荐写法：

```text
研究多模态模型在分布偏移场景下的鲁棒性
要求给出至少 2 个可复现实验
最后输出可投递论文草稿和证据矩阵
```

### 3.2 研究策略

`Research intensity`

- 作用：控制研究节奏和探索强度
- `Light`
  适合先跑通流程 快速试题
- `Balanced`
  默认推荐 大多数情况都先选这个
- `Sprint`
  适合你已经知道题目方向 想多做尝试

`Decision policy`

- 作用：控制关键决策更偏自动还是更偏人工
- `Autonomous`
  适合你希望系统自己推进
- `User gated`
  适合你希望关键点由你确认

`Launch mode`

- 作用：控制从哪种研究模式启动
- `Standard full run`
  从完整流程开始
  不确定时优先选这个
- `Continue existing state`
  已经有旧结果 继续往下跑
- `Review only`
  只做评审 打分 审稿意见整理
- `Rebuttal / revision`
  已有论文草稿或评审意见 主要做修稿和答辩准备

`Auto approve gate`

- 作用：是否自动通过中间 Gate
- `Yes`
  适合个人快速试跑
- `No`
  适合你想保留更多人工确认点

`Objectives`

- 作用：把课题拆成几个明确目标
- 建议填法：每行一个
- 示例：

```text
验证 baseline 的主要弱点
设计 2 个改进方向
形成可复现实验
输出可投论文草稿
```

### 3.3 资料与约束

`Baseline links`

- 作用：告诉系统已有方法和对照对象是什么
- 填什么：
  - 论文链接
  - GitHub 仓库
  - 本地笔记或文件路径
- 建议：每行一个

`References / repos`

- 作用：补充核心参考资料
- 填什么：
  - 重点论文
  - 开源仓库
  - 你想让系统优先借鉴的实现

`Runtime constraints`

- 作用：告诉系统运行边界
- 推荐写法：
  - GPU 型号或显存限制
  - 时间预算
  - 不能联网或必须联网
  - 结果验证要求
- 示例：

```text
本地仅有 8GB 显存
单次实验不超过 2 小时
必须保留可复现脚本和日志
```

## 4. Settings 怎么用

`Settings` 分成 6 个子页：

- `界面`
- `研究`
- `运行`
- `质量`
- `服务`
- `自动化`

### 4.1 界面

`界面语言`

- 作用：切换中文或英文
- 选什么：
  - 中文用户就选 `简体中文`
  - 需要给海外同事演示再切 `English`

`默认项目名`

- 作用：配置默认项目名
- 什么时候填：
  - 你经常复用同一个研究环境时

### 4.2 研究

`默认研究主题`

- 作用：默认研究方向
- 适合：
  - 你长期集中在一个主题上

`研究领域`

- 作用：给系统一个领域边界
- 填法：英文逗号分隔
- 示例：

```text
llm, robustness, multimodal
```

`每日论文数`

- 作用：每天文献筛选数量
- 选什么：
  - 初期探索：`10`
  - 想更广：`20`

`质量阈值`

- 作用：控制研究条目保留标准
- 不确定时可先用默认值

`Baseline 摘要`

- 作用：一句话说明现有 baseline 假设或已知短板
- 示例：

```text
现有 baseline 在 out-of-domain 样本上误判率明显升高
```

### 4.3 运行

`时区`

- 作用：用于调度和日志展示
- 中国用户一般填 `Asia/Shanghai`

`并行任务数`

- 作用：同一时间并发任务数量
- 本地机器建议：
  - 普通笔记本：`1`
  - 算力较强工作站：`2`

`审批超时小时`

- 作用：人工确认等待上限

`重试次数`

- 作用：阶段失败后的自动重试次数
- 一般先用 `0` 或 `1`

`运行位置`

- 作用：控制实验在本地还是云端执行
- `本地机器`
  适合你本地显卡和内存够用
- `本地 Docker`
  适合你希望本地环境更隔离
- `SSH 云服务器`
  适合本地显存不够 内存不够 训练太慢

### 4.4 云服务器运行怎么填

当 `运行位置` 选成 `SSH 云服务器` 时，下面这些字段就需要认真填：

`云服务器地址`

- 填什么：服务器公网 IP 或域名
- 示例：`123.45.67.89`

`登录用户`

- 填什么：SSH 用户名
- 示例：`ubuntu`

`SSH 端口`

- 一般是 `22`
- 如果云厂商改过端口 就填实际值

`SSH 密钥路径`

- 填什么：你本地私钥路径
- Windows 示例：

```text
C:\Users\你的用户名\.ssh\id_rsa
```

`远程工作目录`

- 作用：实验代码和中间文件在云端放哪里
- 推荐：

```text
/tmp/researchclaw_experiments
```

`远程 Python`

- 作用：告诉系统云端用哪个 Python
- 常见填法：
  - `python3`
  - `/home/ubuntu/miniconda3/bin/python`

`远程使用 Docker`

- 什么时候选 `是`
  - 你云端环境杂乱
  - 你想减少依赖污染
- 什么时候选 `否`
  - 你已经手工配好环境

推荐情况：

- 本地 8GB 显存以下
  优先考虑云服务器
- 本地 CPU 内存紧张
  优先考虑云服务器
- 你只是验证流程
  先本地
- 你要正式跑实验
  更建议云服务器

### 4.5 质量

`目标 venue`

- 作用：决定评审风格和质量目标
- 示例：
  - `CCF-A`
  - `NeurIPS`
  - `ICLR`

`目标评分`

- 作用：希望评审循环至少达到的分数
- 一般可先设 `6` 到 `8`

`最大评审轮次`

- 作用：最多允许修稿几轮
- 一般 `3` 到 `4`

`最小提升`

- 作用：每轮评审后最低应提升多少
- 不想让系统空转时很有用

`通知渠道`

- 作用：接收阶段通知
- 示例：
  - `lark`
  - `wecom`
  - `email`
  - `webhook`

### 4.6 服务

`启用 Web 服务`

- 作用：是否开启当前这个可视化界面
- 一般保持开启

`主机地址`

- 本机访问通常填 `127.0.0.1`
- 局域网演示时可改成 `0.0.0.0`

`端口`

- 默认 `8080`
- 被占用时换成 `8090` 之类

`语音入口`

- 作用：是否开启语音能力
- 不需要就关掉

`启用仪表盘`

- 作用：是否保留后台刷新和状态采集
- 一般建议保持开启

### 4.7 自动化

`在 Gate 暂停`

- 作用：到关键门控点时先停下来
- 适合你想保留人工审核

`每阶段暂停`

- 作用：每个阶段都等你确认
- 只建议在调试时使用

`允许分支`

- 作用：允许系统并行尝试不同方向
- 算力紧张时可以关掉

`最大分支数`

- 作用：最多同时尝试几条路线
- 本地建议 `1` 到 `2`
- 云端可提高

`启用多项目`

- 作用：允许同一套系统管理多个课题

`项目目录`

- 作用：所有项目数据落盘位置

`最大并发项目`

- 作用：同时运行的项目数上限
- 本地建议 `1`
- 云端资源足够可以增加

`共享知识`

- 作用：多个项目之间共享部分经验和知识
- 主题接近时建议开启

## 5. 我应该怎么选

### 情况 A：第一次跑通

推荐：

- `Launch mode` 选 `Standard full run`
- `Research intensity` 选 `Balanced`
- `Decision policy` 选 `Autonomous`
- `Auto approve gate` 选 `Yes`
- `运行位置` 选 `本地机器`

### 情况 B：本地显卡不够

推荐：

- `运行位置` 选 `SSH 云服务器`
- 填好 `云服务器地址 / 用户 / 端口 / 密钥路径 / 远程 Python`
- 如果远端环境复杂 再打开 `远程使用 Docker`

### 情况 C：你已经有论文草稿

推荐：

- `Launch mode` 选 `Rebuttal / revision`
- `Objectives` 里直接写修稿目标
- `References / repos` 填审稿意见 论文草稿仓库和关键文档

### 情况 D：你只想让系统帮你审

推荐：

- `Launch mode` 选 `Review only`
- `Decision policy` 可选 `User gated`

## 6. 一条最稳妥的起步路径

1. 先在 `Settings -> 运行` 里确定算力位置
2. 如果本地不够 先把云服务器信息填好
3. 打开 `Start Research`
4. 在 `基本信息` 里写清课题
5. 在 `研究策略` 里选 `Balanced + Standard full run`
6. 在 `资料与约束` 里填 baseline 和参考资料
7. 创建项目
8. 先看 `Details`
9. 再看 `Canvas`
10. 最后在 `Studio` 里观察消息流和时间线

## 7. 常见问题

### 页面能打开 但没有项目

说明你还没创建项目

### 页面卡顿

- 降低并行任务数
- 关闭多分支
- 切到云服务器

### 本地跑不动

优先做这三件事：

1. `运行位置` 改成 `SSH 云服务器`
2. 并行任务数改成 `1`
3. 分支数改小

### 不知道字段怎么填

先按下面这个最小原则：

- 看不懂的字段先保留默认值
- 只认真填 `主题 目标 baseline 参考资料 运行位置`
- 第一次优先把流程跑通 不要一次把所有高级选项都打开
